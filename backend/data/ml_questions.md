# ML Interview Questions Bank

## Category: Fundamentals

### Q1: What's the trade-off between bias and variance?
**Answer:** If our model is too simple and has very few parameters then it may have high bias and low variance. On the other hand if our model has large number of parameters then it's going to have high variance and low bias. So we need to find the right/good balance without overfitting and underfitting the data.
**Field:** general

### Q2: What is gradient descent?
**Answer:** Gradient descent is an optimization algorithm used to find the values of parameters (coefficients) of a function (f) that minimizes a cost function (cost). Gradient descent is best used when the parameters cannot be calculated analytically (e.g. using linear algebra) and must be searched for by an optimization algorithm.
**Field:** general

### Q3: Explain over- and under-fitting and how to combat them?
**Answer:** Underfitting arises when the flexibility of a model is not adequate to capture the underlying pattern in a training dataset. Overfitting arises when the model is too flexible -- the model has memorized the training data. Combat with regularization, cross-validation, early stopping, dropout, data augmentation.
**Field:** general

### Q4: How do you combat the curse of dimensionality?
**Answer:** Feature Selection (manual or statistical), Principal Component Analysis (PCA), Multidimensional Scaling, Locally linear embedding.
**Field:** general

### Q5: What is regularization, why do we use it, and give some examples of common methods?
**Answer:** A technique that discourages learning a more complex or flexible model, so as to avoid the risk of overfitting. Examples: Ridge (L2 norm) shrinks coefficients close to zero but never exactly zero. Lasso (L1 norm) forces some coefficients to be exactly zero when lambda is sufficiently large, performing variable selection and yielding sparse models.
**Field:** general

### Q6: Explain Principal Component Analysis (PCA)?
**Answer:** PCA is a dimensionality reduction technique used to reduce the number of features while retaining as much information as possible. It identifies the directions (principal components) in which the data varies the most, and projects the data onto a lower-dimensional subspace along these directions.
**Field:** general

### Q7: Why is ReLU better and more often used than Sigmoid in Neural Networks?
**Answer:** (1) Computation Efficiency: ReLU is a simple threshold so forward and backward paths are faster. (2) Reduced Likelihood of Vanishing Gradient: Gradient of ReLU is 1 for positive values and 0 for negative, while Sigmoid saturates quickly. (3) Sparsity: When input is negative, ReLU outputs zero, meaning fewer neurons fire and the network is lighter.
**Field:** general

## Category: Computer Vision

### Q8: What is the receptive field of a CNN?
**Answer:** The receptive field is the portion of space within inputs that will be used during an operation to generate an output. For a CNN filter of size k applied to an input, the receptive field grows with each layer depending on kernel size, stride, and dilation.
**Field:** cv

### Q9: How would you remove outliers when trying to estimate a flat plane from noisy samples?
**Answer:** Random sample consensus (RANSAC) is an iterative method to estimate parameters of a mathematical model from observed data that contains outliers, according them no influence on the parameter estimates.
**Field:** cv

### Q10: How does CBIR work?
**Answer:** Content-based image retrieval uses images to gather metadata on their content. Approaches range from feature detection to CNNs extracting dense features associated to a known distribution of keywords. We care about the similarity between metadata generated by a known image and a list of known labels projected into the metadata space.
**Field:** cv

### Q11: Describe how convolution works. What about grayscale vs RGB imagery?
**Answer:** In a CNN, convolution is applied to input using a kernel/filter. The kernel slides over the image in strides, performing element-wise multiplications and summing results to produce a feature map. For RGB, the sliding window becomes a sliding cube. The output shape is determined by kernel size, number of kernels, stride, padding, and dilation.
**Field:** cv

### Q12: How would you create a 3D model of an object from imagery and depth sensor measurements?
**Answer:** Two popular methods: Structure from Motion (SfM) -- better suited for large scenes, and Multi-View Stereo (MVS) -- better suited for small objects.
**Field:** cv

### Q13: What is data normalization and why do we need it?
**Answer:** Data normalization is a preprocessing step to rescale values by subtracting the mean and dividing by standard deviation. Without it, high-magnitude features would be weighted more in the cost function. Normalization makes all features weighted equally for better convergence during backpropagation.
**Field:** general

### Q14: Why do we use convolutions for images rather than just FC layers?
**Answer:** Convolutions preserve, encode, and use spatial information from the image. FC layers would lose relative spatial information. CNNs also have partially built-in translation invariance since each convolution kernel acts as its own filter/feature detector.
**Field:** cv

### Q15: What makes CNNs translation invariant?
**Answer:** Each convolution kernel acts as its own filter/feature detector. For object detection, it doesn't matter where in the image the object is since convolution is applied in a sliding window fashion across the entire image.
**Field:** cv

### Q16: Why do we have max-pooling in classification CNNs?
**Answer:** Max-pooling reduces computation since feature maps are smaller after pooling. You don't lose too much semantic information since you're taking the maximum activation. There's also a theory that max-pooling contributes to giving CNNs more translation invariance.
**Field:** cv

### Q17: Why do segmentation CNNs typically have an encoder-decoder structure?
**Answer:** The encoder CNN acts as a feature extraction network, while the decoder uses that information to predict image segments by decoding the features and upscaling to the original image size.
**Field:** cv

### Q18: What is the significance of Residual Networks?
**Answer:** Residual connections allow direct feature access from previous layers, making information propagation throughout the network much easier. Using local skip connections gives the network an ensemble multi-path structure, giving features multiple paths to propagate.
**Field:** general

### Q19: What is batch normalization and why does it work?
**Answer:** Batch normalization normalizes the inputs of each layer so they have mean output activation of zero and standard deviation of one, computed per mini-batch. This helps with the internal covariate shift problem where the distribution of each layer's inputs changes during training, making learning faster and more stable.
**Field:** general

### Q20: Why use many small convolutional kernels (3x3) rather than few large ones?
**Answer:** From VGGNet: (1) Several smaller kernels achieve the same receptive field as fewer large ones but with fewer parameters and computations. (2) With smaller kernels you use more filters, thus more activation functions, leading to a more discriminative mapping function.
**Field:** cv

### Q21: What is Non-Maximum Suppression (NMS)?
**Answer:** NMS eliminates multiple detections of the same object. Sort bounding boxes by scores (N log N). Starting with the highest score, remove boxes whose IoU overlap exceeds a threshold (N^2). Can be optimized with R-tree or KD-tree for querying overlapping boxes.
**Field:** cv

## Category: Model Evaluation & Training

### Q22: Why do we need a validation set and test set?
**Answer:** Training dataset is used for fitting parameters. Validation dataset measures performance on non-training examples and is used to tune hyperparameters. Test dataset measures performance on previously unseen examples and should only be used once after tuning with the validation set. Using validation repeatedly leaks information.
**Field:** general

### Q23: What is stratified cross-validation?
**Answer:** In stratified cross-validation, the split preserves the ratio of categories in both training and validation datasets. Use when the dataset has multiple categories (especially if small and imbalanced) or data of different distributions.
**Field:** general

### Q24: Why do ensembles typically have higher scores than individual models?
**Answer:** An ensemble combines multiple models that make different errors. Errors of one model are compensated by correct guesses of others. Diversity is achieved by using different ML algorithms, different data subsets (bagging), or iteratively weighting samples by errors (boosting).
**Field:** general

### Q25: What is an imbalanced dataset and how to deal with it?
**Answer:** A dataset with different proportions of target categories. Solutions: oversampling or undersampling, data augmentation in less frequent categories, using appropriate metrics (precision, recall, F-score instead of just accuracy).
**Field:** general

### Q26: Differences between supervised, unsupervised, and reinforcement learning?
**Answer:** Supervised learning trains on labeled data to learn input-output relationships. Unsupervised learning uses only unlabeled data to learn representations. Reinforcement learning has input data and a reward signal; the model learns a policy that maximizes cumulative reward.
**Field:** general

### Q27: What is data augmentation?
**Answer:** Synthesizing new data by modifying existing data while preserving or knowingly changing the target. For images: resize, flip, rotate, add noise, deform, modify colors. Each problem needs customized augmentation (e.g., flips would harm OCR but resizes may help).
**Field:** general

## Category: Metrics

### Q28: What is Precision?
**Answer:** Precision (positive predictive value) is the fraction of relevant instances among retrieved instances. Precision = true positive / (true positive + false positive).
**Field:** general

### Q29: What is Recall?
**Answer:** Recall (sensitivity) is the fraction of relevant instances that have been retrieved over the total amount of relevant instances. Recall = true positive / (true positive + false negative).
**Field:** general

### Q30: Define F1-score.
**Answer:** The weighted average of precision and recall. F1-Score = 2 * (precision * recall) / (precision + recall). It considers both false positives and false negatives.
**Field:** general

### Q31: How does a ROC curve work?
**Answer:** The ROC curve is a graphical representation of the contrast between true positive rates and false positive rates at various thresholds. It represents the trade-off between model sensitivity (true positives) and the probability of triggering false alarms (false positives).
**Field:** general

### Q32: Difference between Type I and Type II error?
**Answer:** Type I error is a false positive (claiming something happened when it hasn't). Type II error is a false negative (claiming nothing is happening when something is).
**Field:** general

### Q33: Difference between generative and discriminative models?
**Answer:** A generative model learns categories of data (models P(X|Y) and P(Y)). A discriminative model learns the distinction between categories (models P(Y|X) directly). Discriminative models generally outperform generative models on classification tasks.
**Field:** general

## Category: Deep Learning

### Q34: What is the cost function?
**Answer:** Cost function is a scalar function which quantifies the error factor of the Neural Network. Lower the cost function, better the Neural network. It measures the difference between predicted and actual outputs.
**Field:** general

### Q35: List different activation functions.
**Answer:** Linear Neuron, Binary Threshold Neuron, Stochastic Binary Neuron, Sigmoid Neuron, Tanh function, Rectified Linear Unit (ReLU), Leaky ReLU, ELU, Swish, GELU.
**Field:** general

### Q36: What is Learning Rate?
**Answer:** Learning rate is a hyper-parameter that controls how much we are adjusting the weights of our network with respect to the loss gradient. Too high causes divergence, too low causes slow convergence.
**Field:** general

### Q37: What is Momentum in NN optimization?
**Answer:** Momentum lets the optimization algorithm remember its last step, and adds some proportion of it to the current step. Even if stuck in a flat region or small local minimum, it can escape and continue towards the true minimum.
**Field:** general

### Q38: Difference between Batch Gradient Descent and SGD?
**Answer:** Batch gradient descent computes the gradient using the whole dataset -- good for convex/smooth error manifolds. SGD computes the gradient using a single sample -- works better for error manifolds with many local maxima/minima, as noisy gradients help escape local minima.
**Field:** general

### Q39: Epoch vs Batch vs Iteration?
**Answer:** Epoch: one forward pass and one backward pass of all training examples. Batch: examples processed together in one pass. Iteration: number of training examples / Batch size.
**Field:** general

### Q40: What is vanishing gradient?
**Answer:** As we add more hidden layers, backpropagation becomes less useful in passing information to lower layers. The gradients begin to vanish and become very small relative to the weights, making deep networks hard to train.
**Field:** general

### Q41: What are dropouts?
**Answer:** Dropout is a regularization technique that prevents overfitting by randomly dropping out (setting to zero) some units in a neural network during training. This prevents co-adaptation of features and acts as an ensemble of many sub-networks.
**Field:** general

### Q42: Define LSTM.
**Answer:** Long Short Term Memory is a type of recurrent neural network explicitly designed to address the long-term dependency problem. It maintains a cell state with gates (forget, input, output) that control what information to remember and forget.
**Field:** nlp

### Q43: List the key components of LSTM.
**Answer:** Gates (forget gate, input/memory gate, output/read gate), tanh activation (values between -1 to 1), sigmoid activation (values between 0 to 1), cell state (long-term memory), hidden state (short-term memory).
**Field:** nlp

### Q44: List variants of RNN.
**Answer:** LSTM (Long Short Term Memory), GRU (Gated Recurrent Unit), Bidirectional RNN, Deep RNN, End to End Network, Memory Network.
**Field:** nlp

### Q45: What is an Autoencoder?
**Answer:** Autoencoder is a neural network used to learn compressed representations of input data. Applications: data denoising, dimensionality reduction, image reconstruction, image colorization, anomaly detection.
**Field:** general

### Q46: What are the components of GAN?
**Answer:** Generator (creates fake samples to fool the discriminator) and Discriminator (distinguishes between real and generated samples). They are trained adversarially in a minimax game.
**Field:** general

### Q47: Difference between boosting and bagging?
**Answer:** Both are ensemble techniques. Bagging takes bootstrap samples (with replacement), each training a weak learner independently (parallel). Boosting uses all data, but instances misclassified by previous learners get more weight, so subsequent learners focus on hard examples (sequential).
**Field:** general

## Category: Dimensionality Reduction

### Q48: Difference between LDA and PCA?
**Answer:** Both are linear transformation techniques. LDA is supervised and attempts to find a feature subspace that maximizes class separability. PCA is unsupervised and finds directions of maximal variance, ignoring class labels.
**Field:** general

### Q49: What is t-SNE?
**Answer:** t-Distributed Stochastic Neighbor Embedding is an unsupervised, non-linear technique primarily for data exploration and visualizing high-dimensional data. It preserves local similarities (small pairwise distances) in the low-dimensional projection.
**Field:** general

### Q50: Difference between t-SNE and PCA?
**Answer:** PCA is linear, seeks to maximize variance, and preserves large pairwise distances. t-SNE preserves only small pairwise distances (local similarities). PCA can lead to poor visualization for non-linear manifold structures; t-SNE handles non-linear structure better.
**Field:** general

### Q51: What is UMAP?
**Answer:** Uniform Manifold Approximation and Projection is a manifold learning technique for dimension reduction based on Riemannian geometry and algebraic topology. It is faster and more scalable than t-SNE and often better at preserving global structure.
**Field:** general

### Q52: Difference between t-SNE and UMAP?
**Answer:** UMAP is often better at preserving global structure compared to t-SNE, meaning inter-cluster relations are potentially more meaningful. UMAP is also significantly faster. However, both warp high-dimensional shape when projecting to lower dimensions.
**Field:** general

## Category: General ML Concepts

### Q53: Instance-Based vs Model-Based Learning?
**Answer:** Instance-based Learning: the system learns examples by heart, then generalizes using a similarity measure. Model-based Learning: build a model from examples, then use that model to make predictions.
**Field:** general

### Q54: When to use Label Encoding vs One Hot Encoding?
**Answer:** Use One-Hot Encoding when the categorical feature is not ordinal and the number of categories is small. Use Label Encoding when the feature is ordinal (e.g., Jr. kg, Sr. kg, High school) or the number of categories is large (one-hot would cause high memory consumption).
**Field:** general

### Q55: Difference between Bayesian and frequentist statistics?
**Answer:** Frequentist statistics estimates population parameters using sample statistics, providing point estimates and confidence intervals. Bayesian statistics uses prior knowledge to update beliefs about parameters, providing probability distributions. Bayesian incorporates prior knowledge; frequentist doesn't.
**Field:** general

### Q56: Basic difference between LSTM and Transformers?
**Answer:** LSTMs consist of RNN cells designed to store and manipulate information across time steps sequentially. Transformers contain stacks of encoder and decoder layers with self-attention mechanisms and feed-forward networks, processing all positions in parallel. Transformers scale better and capture long-range dependencies more effectively.
**Field:** nlp

### Q57: What are RCNNs?
**Answer:** Region-based Convolutional Neural Networks are designed for object detection in computer vision. The RCNN approach combines region proposal techniques with CNNs to identify and locate objects within an image. Variants include Fast R-CNN, Faster R-CNN, and Mask R-CNN.
**Field:** cv

## Category: NLP

### Q58: Difference between stemming and lemmatization?
**Answer:** Both reduce words to their base form. Stemming is a crude heuristic that chops off word endings (fast but less accurate). Lemmatization uses vocabulary and morphological analysis to find the proper base form (more accurate but slower). Example with "better": stemming gives "better", lemmatization gives "good".
**Field:** nlp

### Q59: What is Latent Semantic Indexing (LSI)?
**Answer:** LSI extracts underlying meaning from text documents using mathematical techniques like SVD to identify patterns in word co-occurrence. Based on the idea that words used in similar contexts tend to have similar meanings.
**Field:** nlp

### Q60: What is Dependency Parsing?
**Answer:** Dependency parsing analyzes grammatical structure of a sentence and identifies relationships between words. It builds a directed graph where words are nodes and grammatical relationships are edges. Each node has one parent and can have multiple children.
**Field:** nlp

### Q61: Name different approaches for text summarization.
**Answer:** Extractive summarization (selects important sentences), Abstractive summarization (generates new sentences), LSA-based, LDA-based, and Neural-based summarization (using deep neural networks like transformers).
**Field:** nlp

### Q62: Explain what is an n-gram model.
**Answer:** A statistical language model where the probability of a word depends on the n-1 preceding words. The model represents text as sequences of n consecutive words and uses frequency in a large corpus to estimate probabilities.
**Field:** nlp

### Q63: How does TF-IDF measure word importance?
**Answer:** TF-IDF = Term Frequency x Inverse Document Frequency. TF = number of times word appears in a document normalized by total words. IDF = logarithm of (total documents / documents containing the word). Words frequent in a document but rare across corpus get high scores.
**Field:** nlp

### Q64: What is perplexity used for?
**Answer:** Perplexity evaluates the quality of probability/language models. It quantifies uncertainty when predicting the next word. Lower perplexity means better model. Formula: Perplexity = 2^(entropy of the distribution).
**Field:** nlp

### Q65: What is the Bag-of-Words model?
**Answer:** A text representation where text is represented as a bag (multiset) of its words, disregarding grammar and word order but keeping track of word frequency. Simple and efficient but discards grammatical information and word order.
**Field:** nlp

### Q66: How does the Markov assumption affect the bi-gram model?
**Answer:** The Markov assumption states the probability of a word depends only on the preceding word. It simplifies the bi-gram model by reducing variables, making it computationally efficient, but limits context which can lead to probability estimation errors. Increasing n-gram order increases context.
**Field:** nlp

### Q67: What are the most common word embedding methods?
**Answer:** Count-based methods (e.g., LSA), Prediction-based methods (e.g., Word2Vec, CBOW), Hybrid methods (e.g., GloVe), Neural Language Model based (e.g., BERT, GPT). Modern approaches use contextual embeddings from transformers.
**Field:** nlp

### Q68: First steps before applying an NLP algorithm to a corpus?
**Answer:** Text pre-processing (remove special characters, lowercasing, stop words), Tokenization (word/sentence/subword), Text normalization (stemming/lemmatization), Feature extraction (vocabulary, term-document matrix), Data splitting (train/val/test), Annotation (POS tagging, NER).
**Field:** nlp

### Q69: Types of linguistic ambiguities?
**Answer:** Lexical ambiguity (word has multiple meanings), Syntactic ambiguity (sentence parsed multiple ways), Semantic ambiguity (multiple meanings in context), Pragmatic ambiguity (different interpretations based on intent), Anaphora resolution, Homonymy, Polysemy.
**Field:** nlp

### Q70: What is Retrieval-Augmented Generation (RAG)?
**Answer:** RAG combines retrieval-based and generative approaches. It retrieves relevant documents/passages from a knowledge base using embeddings and similarity search, then feeds them as context to a language model to generate grounded, factually accurate responses. Key components: document chunking, embedding generation, vector store, retrieval, and LLM generation.
**Field:** nlp

### Q71: What are different chunking strategies in RAG?
**Answer:** Fixed-size chunking (split by character/token count), Recursive character splitting, Sentence-based splitting, Semantic chunking (group by meaning), Document-structure based (headers, paragraphs), Sliding window with overlap. Choice depends on document type and retrieval granularity needed.
**Field:** nlp

### Q72: What is HNSW (Hierarchical Navigable Small World)?
**Answer:** HNSW is an approximate nearest neighbor search algorithm used in vector databases. It builds a multi-layer graph where upper layers have long-range connections for fast navigation and lower layers have short-range connections for precise search. Offers logarithmic search complexity with high recall.
**Field:** nlp

### Q73: What is Direct Preference Optimization (DPO)?
**Answer:** DPO is an alternative to RLHF for aligning language models with human preferences. Instead of training a separate reward model, DPO directly optimizes the policy using preference pairs (preferred vs rejected responses). It's simpler, more stable, and computationally cheaper than PPO-based RLHF.
**Field:** nlp

### Q74: Difference between RAG and fine-tuning?
**Answer:** RAG retrieves external knowledge at inference time without modifying model weights -- good for factual, up-to-date information. Fine-tuning updates model weights on domain-specific data -- good for learning style, format, or domain-specific reasoning. RAG is cheaper, more flexible, but adds latency. Fine-tuning is permanent but requires retraining for updates.
**Field:** nlp

### Q75: What is YOLOv8 and how does it work?
**Answer:** YOLOv8 is a real-time object detection model that processes the entire image in a single forward pass. It uses a CSPDarknet backbone for feature extraction, a Feature Pyramid Network for multi-scale feature fusion, and decoupled detection heads for classification and bounding box regression. It's anchor-free and uses task-aligned assignment for training.
**Field:** cv

### Q76: What is Real-ESRGAN?
**Answer:** Real-ESRGAN is a super-resolution model for enhancing image quality. It extends ESRGAN with a high-order degradation modeling process that simulates real-world image degradation (blur, noise, compression artifacts). Uses a U-Net discriminator with spectral normalization for more stable training.
**Field:** cv

### Q77: What is Entity-Context Divergence in RAG?
**Answer:** Entity-Context Divergence (ECD) is a metric that quantifies factual inconsistencies in RAG systems by measuring the divergence between entity representations in retrieved contexts versus the model's internal knowledge. High ECD indicates potential hallucination or knowledge conflicts.
**Field:** nlp

### Q78: What is mAP (mean Average Precision) in object detection?
**Answer:** mAP is the primary evaluation metric for object detection. It computes the average precision (area under the precision-recall curve) for each class, then averages across all classes. mAP@0.5 uses IoU threshold of 0.5; mAP@0.5:0.95 averages across multiple IoU thresholds.
**Field:** cv

### Q79: What are Vision Transformers (ViT)?
**Answer:** Vision Transformers apply the transformer architecture to image classification by splitting images into fixed-size patches, linearly embedding them, adding positional embeddings, and processing through standard transformer encoder blocks. They achieve competitive or superior performance to CNNs when pre-trained on large datasets.
**Field:** cv

### Q80: What is transfer learning?
**Answer:** Transfer learning uses a model trained on one task as the starting point for a model on a different but related task. Common approaches: feature extraction (freeze pretrained layers, train new head) and fine-tuning (unfreeze some/all layers and train with small learning rate). Reduces training time and data requirements.
**Field:** general
